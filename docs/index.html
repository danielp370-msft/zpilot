zpilot

Mission control for AI coding sessions. Monitor, manage, and orchestrate multiple terminal multiplexer sessions from a single pane of glass.

101 tests passing 11 MCP tools 8 REST endpoints Python 3.11+

Overview

zpilot is a from-scratch mission control system for monitoring multiple AI coding sessions (e.g. copilot-cli, aider, or any terminal process) running inside Zellij terminal multiplexer panes.

It wraps Zellij with a rich instrumentation layer — real-time output logging, FIFO-based command injection, state detection (active / idle / waiting / error), pluggable notifications, and an MCP server that lets AI agents monitor and control sessions programmatically.

💡 Why zpilot?
When you're running 5+ AI coding agents in parallel, you need a way to know which ones are waiting for input, which ones hit an error, and which ones are still crunching. zpilot gives you that visibility — in a web dashboard, a terminal TUI, or through an MCP interface for AI-to-AI orchestration.

Data Flow

Each Zellij pane runs a shell wrapper (shell_wrapper.py) that forks a PTY, logging all output to /tmp/zpilot/logs/ and accepting commands via a named FIFO at /tmp/zpilot/fifos/. The daemon polls these logs, feeds content to the detector state machine, and emits state-change events to a JSONL file. The web dashboard and TUI subscribe to these events via SSE or file tailing.

Quick Start

1. Create a monitored session

zpilot new my-agent "echo Hello from zpilot"

2. Launch the web dashboard

zpilot web --port 8097

Open http://localhost:8097 in your browser.

3. Start the background daemon

zpilot daemon &

4. Check status from the command line

zpilot status

# ⏳ my-agent     active  (idle: 0s)  │ echo Hello from zpilot
# 🔔 demo-build   waiting (idle: 12s) │ danielp@host:~/project$
# ✅ demo-copilot idle    (idle: 45s) │ danielp@host:~$

5. Connect an AI agent via MCP

zpilot serve   # starts MCP server on stdio

Configuration

Stored at ~/.config/zpilot/config.toml. Run zpilot config to view or create the default file.

Full Config Reference

# ~/.config/zpilot/config.toml

[general]
poll_interval = 5          # seconds between daemon polls
idle_threshold = 30        # seconds of no output → idle
events_file = "/tmp/zpilot/events.jsonl"

[detection]
bel_enabled = true         # detect terminal bell (\x07)
prompt_patterns = [
    '^\$ $',              # bare dollar prompt
    '^> $',               # chevron prompt
    '^❯ ',               # starship/zsh prompt
    '\$\s*$',             # line ending with $ (user@host:path$)
    '#\s*$',              # root prompt
    '>>>\s*$',            # Python REPL
    '\.\.\.\s*$',         # Python continuation
]
error_patterns = [
    '^Error:',
    '^FATAL:',
    '^panic:',
]

[notifications]
enabled = true
adapter = "log"           # "log", "ntfy", "desktop", "webhook"
notify_on = ["waiting", "error", "exited"]

[notifications.ntfy]
topic = "zpilot"
server = "https://ntfy.sh"
⚠️ TOML String Escaping
TOML single-quoted strings ('...') are literal — no escape processing. Use single backslashes for regex: '\$\s*$' not '\\$\\s*$'.

Detection Engine

The PaneDetector is the brain of zpilot — a state machine that analyzes terminal content via content hashing, pattern matching, and timing.

Detection Priority (first match wins)

#CheckConditionReturns
1Error patternsLast 5 lines match ^Error:, ^FATAL:, ^panic:ERROR
2Prompt patternsLast 2 lines match prompt regex AND stable ≥ 1sWAITING
3BEL character\x07 in content (terminal bell)WAITING
4Content changingHash changed OR idle < 3sACTIVE
5Idle thresholdNo change ≥ idle_threshold secondsIDLE
6FallbackBetween active and idleACTIVE

Content Hashing

Each call computes an MD5 hash of ANSI-stripped content. If the hash differs from the previous call, content has changed. The detector tracks _last_change_time per pane to compute idle seconds.

Full-Screen App Handling

Apps like top and vim use alternate screen buffers and clear-screen sequences. The ANSI stripper detects these (\x1b[?1049h, \x1b[2J) and keeps only the last frame, preventing infinite scroll in the web viewer.

Shell Wrapper (PTY Bridge)

shell_wrapper.py runs inside each Zellij pane, providing two capabilities that Zellij alone doesn't offer:

📝 Output Logging

All output logged to /tmp/zpilot/logs/<session>--main.log. Readable without an attached terminal.

📥 FIFO Injection

Named FIFO at /tmp/zpilot/fifos/<session>.fifo accepts commands injected into PTY stdin.

Internal Flow

1. zellij --session my-session    # create Zellij session
2. python3 shell_wrapper.py my-session
   ├── mkfifo /tmp/zpilot/fifos/my-session.fifo
   ├── pty.fork()                   # fork pseudo-terminal
   │   └── child: exec($SHELL)     # run user's shell
   └── parent:
       ├── Thread: FIFO reader → PTY stdin
       ├── select(): PTY stdout → log file + stdout
       └── select(): real stdin → PTY stdin

File Locations

/tmp/zpilot/
├── logs/
│   └── my-session--main.log   # real-time output
├── fifos/
│   └── my-session.fifo        # command injection
└── events.jsonl               # event bus

Event System

A file-based event bus using append-only JSONL at /tmp/zpilot/events.jsonl. Zero dependencies, easy to debug with tail -f.

Event Structure

{
  "timestamp": 1709734567.89,
  "event_type": "state_change",
  "session": "demo-build",
  "pane": "main",
  "old_state": "active",
  "new_state": "waiting",
  "details": "Prompt detected"
}

Event Types

TypeWhen
state_changePane transitions between states
notificationNotification sent to user
errorInternal error

EventBus API

bus = EventBus("/tmp/zpilot/events.jsonl")

bus.emit(event)              # append to file
bus.recent(n=50)             # last N events
bus.all_events()             # read all
bus.clear()                  # truncate
bus.on_event(callback)       # register listener
await bus.tail(callback)     # async tail -f

Four-Tier Dump Strategy

dump_pane() tries four methods to read pane content:

  1. Log file — /tmp/zpilot/logs/ (fastest, works headless)
  2. Session logs — Zellij's own log directory
  3. Dump screen — zellij action dump-screen
  4. Probe — Send echo command and read back (last resort)

Troubleshooting

Sessions always show "ACTIVE"

Check prompt patterns in ~/.config/zpilot/config.toml. TOML single-quoted strings don't process escapes — use '\$\s*$' (single backslash), not '\\$\\s*$'.

Can't type in the web dashboard

Click directly on the terminal area (dark background). The inline prompt appears on focus. Toggle the bottom input bar with the ⌨️ button.

New sessions show Zellij tips screen

Session creation has a timing-sensitive step dismissing the "About" pane. Kill and recreate: zpilot new <name>.

top/htop causes scrolling

Ensure you have the latest _strip_ansi() with alt-screen-buffer detection.

WSL: Can't reach localhost

hostname -I | awk '{print $1}'
# Access at http://<WSL-IP>:8097
# Or add networkingMode=mirrored to %USERPROFILE%\.wslconfig
